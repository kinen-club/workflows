# Kinen Core Methodology

**Load the full methodology from:** `methodology/core.md`

## Quick Reference

When using kinen:

1. **Start**: Create session folder `sessions/YYYYMMDD-NN-name/` with `init.md`
2. **Rounds**: Progress through `rounds/01-foundation.md`, `02-topic.md`, etc.
3. **Living Doc**: Update `artifacts/technical-spec.md` (or appropriate type) after each round
4. **Summarize**: Create `session-summary.md` with all decisions

## Session Types

| Type | Living Document |
|------|-----------------|
| Architecture | `artifacts/technical-spec.md` |
| Implementation | `artifacts/implementation-plan.md` |
| Writing | `artifacts/outline.md` |

## Core Principles

- **Thinking partner**: Challenge assumptions, provide alternatives
- **One round, one focus**: Don't cover everything at once
- **Decisions are explicit**: Document with rationale
- **Progressive refinement**: Each round goes deeper
